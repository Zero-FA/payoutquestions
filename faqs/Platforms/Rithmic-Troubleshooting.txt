Rithmic Troubleshooting (Lag + Error Messages)

Overview
Connection problems with RTrader or Rithmic when using NinjaTrader can interrupt trading. This guide provides troubleshooting steps to identify and resolve common connection issues such as lag, login errors, cache problems, or software configuration issues.


Why Connection Issues Occur

Connection problems may occur due to several common factors including:

- Incorrect login credentials
- Software configuration issues
- Cache or database errors
- Network latency or internet lag
- Outdated software installations
- Insufficient computer performance

Following the troubleshooting steps below can help identify and resolve the issue.


Basic Troubleshooting Steps

Both RTrader and NinjaTrader require correct configuration to establish a stable connection.


Clear Cache and Restart Applications

If connection problems persist, clearing the NinjaTrader cache and restarting both applications may resolve the issue.

Close RTrader
Exit the RTrader application completely.

Clear NinjaTrader Cache

Navigate to the following folder:

Documents → NinjaTrader 8 → db

Delete the folder named:

cache

Restart Applications

Restart both NinjaTrader and RTrader after clearing the cache.


Reinstall Software if Necessary

If the problem continues after clearing cache and restarting:

Uninstall both RTrader and NinjaTrader.

Then reinstall the latest versions from their official websites.

Follow the installation instructions carefully to ensure proper setup.


Lag and Performance Issues

Lag can negatively impact trading performance. Monitoring and resolving lag issues helps maintain stable data flow.


Using the NT8 Lag Analyzer

Rithmic users can measure internet latency using the NT8 Lag Analyzer tool.

Step 1
Download the NT8 Lag Analyzer tool.

Step 2
Import the Add-On

Open NinjaTrader.

Go to:

Tools → Import → NinjaScript Add-On

Select the downloaded ZIP file without extracting it.

Step 3
Configure Market Analyzer

Go to:

New → Market Analyzer

Right-click a blank area and select Columns.

Locate the NeuroStreet folder.

Select:

NS_DataLagTimer

Click OK.

Step 4
Add Instruments

Add the instruments you want to monitor in the Market Analyzer window.


Fixing Lagging Data

If you notice delayed market data or lag, try the following steps.


Reset NinjaTrader Database

Go to:

Tools → Database → Reset Database


Recreate the Rithmic Connection

Delete your existing Rithmic connection.

Add a new connection and reconnect.


Alternative Data Feed

Some traders choose to use IQFeed for futures market data because it may provide faster and more stable data connections.


Clear NinjaTrader Cache Again

Shut down NinjaTrader.

Navigate to:

Documents → NinjaTrader 8 → db

Delete the cache folder again.

Restart NinjaTrader.


Open NinjaTrader in Safe Mode

Hold the CTRL key while launching NinjaTrader.

This starts NinjaTrader in Safe Mode.

Verify Safe Mode is active in the Help menu.

Close NinjaTrader and restart normally afterward.


Disable Tick Refresh on Charts

Open your workspace.

Disconnect from data.

Remove tick refresh from all charts.

Save the workspace.

Reconnect to data and reload the charts.


Check System Performance

Ensure your computer meets the recommended performance requirements.

A system benchmark of approximately 30k or higher is recommended for stable performance.

Verify that your system performance meets the requirements before trading.


When to Contact Support

If you have completed all troubleshooting steps and the issue persists:

Consider uninstalling and reinstalling both RTrader and NinjaTrader.

Review the step-by-step platform setup instructions in your Apex Member Dashboard.


Platform-Specific Support

If you are using trading platforms other than NinjaTrader, contact the support team for that specific platform to check for available plugins or additional troubleshooting guidance.


Conclusion

Connection issues between RTrader, Rithmic, and NinjaTrader can usually be resolved through systematic troubleshooting.

Ensuring correct installation, configuration, and regular system maintenance will help prevent disruptions and maintain a reliable trading environment.

Regularly monitor your connection performance and address lag issues quickly to ensure smooth trading operations.
